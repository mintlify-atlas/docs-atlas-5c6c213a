## Requirements

Exchanges Metadata requires Python 3.11 or higher. The toolkit has been tested on Python 3.11, 3.13, and 3.14.

## Install dependencies

<Steps>
  <Step title="Clone or download the source files">
    Ensure you have the source Python files in your working directory:
    
    - `sgx_company_list.py`
    - `hkex_company_list.py`
    - `bloomberg_sp_company_list.py`
    - `load_exchange_metadata_mysql.py`
    - `requirements.txt`
  </Step>

  <Step title="Install Python packages">
    Install all required dependencies using pip:

    ```bash
    pip install -r requirements.txt
    ```

    This will install the following packages:

## Optional configuration

### Environment variables

You can optionally set environment variables for configuration:

```bash
# HKEX webhook URL (optional)
export HKEX_STOCKLIST_WEBHOOK_URL="https://your-webhook-url.com/endpoint"
```

If not set, the HKEX fetcher will use the default webhook URL.

### Database connection

For MySQL integration, you'll need connection details:

<CodeGroup>

```python Python code
from sqlalchemy import create_engine

# Format: mysql+pymysql://user:password@host:port/database
db_url = "mysql+pymysql://username:password@localhost:3306/marketdata?charset=utf8mb4"
engine = create_engine(db_url)
```

```bash Command line
# Or pass via command line argument
python load_exchange_metadata_mysql.py --db-url "mysql+pymysql://user:pass@host:3306/db"
```

</CodeGroup>

<Warning>
  Ensure your MySQL user has CREATE, INSERT, and DROP permissions on the target database if using replace mode.
</Warning>

## Database setup

If you plan to use the MySQL loading functionality, create the target database:

```sql
CREATE DATABASE IF NOT EXISTS MarketDataHub CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

The loader will automatically create tables:

- `companies_xses` - SGX company listings
- `companies_hkex` - HKEX company listings  
- `companies_bloomberg_equities` - Bloomberg enrichment data

## Troubleshooting

<AccordionGroup>
  <Accordion title="SSL certificate errors">
    If you encounter SSL certificate verification errors, ensure `certifi` is up to date:

    ```bash
    pip install --upgrade certifi
    ```
  </Accordion>

  <Accordion title="MySQL connection issues">
    Verify your connection string format and ensure the MySQL server is accessible:

    ```python
    from sqlalchemy import create_engine

    # Test connection
    engine = create_engine("mysql+pymysql://user:pass@host:3306/db")
    with engine.connect() as conn:
        result = conn.execute("SELECT 1")
        print("Connected successfully!")
    ```
  </Accordion>

  <Accordion title="Import errors">
    If you see `ModuleNotFoundError`, ensure you're in the correct directory with all source files and dependencies are installed:

    ```bash
    # Verify all source files exist
    ls -la *.py

    # Reinstall dependencies
    pip install -r requirements.txt --force-reinstall
    ```
  </Accordion>

  <Accordion title="Rate limiting errors">
    If you encounter rate limiting from SGX or HKEX APIs, adjust the delay parameters:

    ```python
    # Increase delay between requests
    companies = fetcher.enrich_with_sector_details(
        companies,
        delay_per_request=0.2,  # Increase from default 0.05
        max_workers=3  # Reduce from default 5
    )
    ```
  </Accordion>
</AccordionGroup>
