The Exchanges Metadata system stores company data in three primary MySQL tables. Each table has a common `fetched_at_utc` timestamp column that tracks when the data was retrieved.

## Common Fields

All tables include the following system-generated field:

<ResponseField name="fetched_at_utc" type="string">
  ISO 8601 UTC timestamp indicating when the data was fetched and loaded into the database. Automatically added by the `write_table()` function.
</ResponseField>

## companies_xses

Stores company metadata from the Singapore Exchange (SGX/XSES).

### Table Name

```python
DEFAULT_TABLES["sgx"] = "companies_xses"
```

### Schema

<ResponseField name="symbol" type="string">
  Company trading symbol on SGX (e.g., "D05", "C38U"). Primary identifier.
</ResponseField>

<ResponseField name="name" type="string">
  Full legal or trading name of the company.
</ResponseField>

<ResponseField name="exchange" type="string">
  Exchange identifier (e.g., "SGX").
</ResponseField>

<ResponseField name="exchange_country_code" type="string">
  Country code for the exchange.
</ResponseField>

<ResponseField name="market_cap" type="float">
  Market capitalization in SGD.
</ResponseField>

<ResponseField name="pe_ratio" type="float">
  Price-to-earnings ratio.
</ResponseField>

<ResponseField name="yield_pct" type="float">
  Dividend yield percentage.
</ResponseField>

<ResponseField name="price_to_book_ratio" type="float">
  Price-to-book ratio.
</ResponseField>

<ResponseField name="currency" type="string">
  Trading currency code (e.g., "SGD").
</ResponseField>

<ResponseField name="industry" type="string">
  Industry/sector classification from screener. Enriched with detailed sectorName from snapshot reports.
</ResponseField>

<ResponseField name="sector" type="string">
  Detailed industryName from snapshot reports (populated after enrichment).
</ResponseField>

<ResponseField name="sales_ttm" type="float">
  Trailing twelve months sales.
</ResponseField>

<ResponseField name="net_profit_margin" type="float">
  Net profit margin percentage.
</ResponseField>

<ResponseField name="return_on_equity" type="float">
  Return on average common equity percentage.
</ResponseField>

<ResponseField name="price_to_cashflow" type="float">
  Price-to-cash-flow-per-share ratio.
</ResponseField>

<ResponseField name="debt_to_equity" type="float">
  Total debt to total equity ratio.
</ResponseField>

<ResponseField name="sales_change_pct" type="float">
  Sales percentage change.
</ResponseField>

<ResponseField name="four_week_change_pct" type="float">
  4-week price percentage change.
</ResponseField>

<ResponseField name="thirteen_week_change_pct" type="float">
  13-week price percentage change.
</ResponseField>

<ResponseField name="twenty_six_week_change_pct" type="float">
  26-week price percentage change.
</ResponseField>

<ResponseField name="fifty_two_week_change_pct" type="float">
  52-week price percentage change.
</ResponseField>

<ResponseField name="is_suspended" type="boolean">
  Whether trading is suspended. Suspended companies are filtered out by default.
</ResponseField>

<ResponseField name="listing_category" type="string">
  "Primary Listing" if exchange is present, otherwise null.
</ResponseField>

<ResponseField name="fetched_at_utc" type="string">
  ISO timestamp when data was fetched (added by write_table).
</ResponseField>

### Data Source

Fetched using `fetch_sgx_companies()` from `SGXCompanyFetcher`.

### Enrichment

When `enrich=True`:
- Calls `enrich_with_sector_details()` with 5 concurrent workers
- Fetches snapshot reports to populate `sector` field with detailed industryName
- Updates `industry` field with detailed sectorName
- Delay: 0.05 seconds per request

## companies_hkex

Stores company metadata from the Hong Kong Exchange (HKEX).

### Table Name

```python
DEFAULT_TABLES["hkex"] = "companies_hkex"
```

### Schema

<ResponseField name="symbol" type="string">
  Company stock code on HKEX (e.g., "0001", "0700"). Primary identifier.
</ResponseField>

<ResponseField name="name" type="string">
  Full company name as registered on HKEX.
</ResponseField>

<ResponseField name="ric" type="string">
  Reuters Instrument Code (e.g., "0001.HK").
</ResponseField>

<ResponseField name="last_price" type="float">
  Last traded price in HKD.
</ResponseField>

<ResponseField name="market_cap" type="float">
  Market capitalization in HKD (calculated from mktcap value and mktcap_u unit).
</ResponseField>

<ResponseField name="pe_ratio" type="float">
  Price-to-earnings ratio.
</ResponseField>

<ResponseField name="yield_pct" type="float">
  Dividend yield percentage.
</ResponseField>

<ResponseField name="price_change_pct" type="float">
  Price change percentage.
</ResponseField>

<ResponseField name="net_change" type="float">
  Net price change in HKD.
</ResponseField>

<ResponseField name="currency" type="string">
  Trading currency code (typically "HKD").
</ResponseField>

<ResponseField name="asset_subtype" type="string">
  Asset subtype (e.g., "ODSH" for ordinary shares).
</ResponseField>

<ResponseField name="is_suspended" type="boolean">
  Whether trading is suspended. Suspended companies are filtered out by default.
</ResponseField>

<ResponseField name="industry" type="string">
  HSIC industry classification (populated after enrichment via `enrich_with_quote_details`).
</ResponseField>

<ResponseField name="sector" type="string">
  HSIC sub-sector classification (populated after enrichment).
</ResponseField>

<ResponseField name="listing_category" type="string">
  Listing category (populated after enrichment). "Trading Only" listings are filtered out.
</ResponseField>

<ResponseField name="listing_date" type="string">
  Date of listing on HKEX (populated after enrichment).
</ResponseField>

<ResponseField name="isin" type="string">
  International Securities Identification Number (populated after enrichment).
</ResponseField>

<ResponseField name="lot" type="string">
  Board lot size for trading (populated after enrichment).
</ResponseField>

<ResponseField name="fetched_at_utc" type="string">
  ISO timestamp when data was fetched (added by write_table).
</ResponseField>

### Data Source

Fetched using `fetch_hkex_companies()` from `HKEXCompanyFetcher`.

### Enrichment

When `enrich=True`:
- Calls `enrich_with_quote_details()` with 8 concurrent workers
- Adds industry, sector, listing_category, listing_date, isin, and lot fields
- Delay: 0.03 seconds per request

### Filtering Rules

1. Suspended companies are excluded (`include_suspended=False`)
2. If industry data is available, companies without industry are excluded
3. If listing category data is available, `listing_category="Trading Only"` companies are excluded

## companies_bloomberg_equities

Stores Bloomberg background data for companies from both SGX and HKEX exchanges.

### Table Name

```python
DEFAULT_TABLES["bloomberg_equities"] = "companies_bloomberg_equities"
```

### Schema

<ResponseField name="symbol" type="string">
  Company trading symbol from the source exchange (SGX or HKEX).
</ResponseField>

<ResponseField name="name" type="string">
  Company name from the source exchange.
</ResponseField>

<ResponseField name="mic" type="string">
  Market Identifier Code:
  - `"XSES"` for Singapore Exchange companies
  - `"XHKG"` for Hong Kong Exchange companies
</ResponseField>

<ResponseField name="bloomberg_ticker" type="string">
  Bloomberg ticker format (e.g., "D05:SP", "0700:HK").
</ResponseField>

<ResponseField name="source_url" type="string">
  Bloomberg API endpoint URL used to fetch the data.
</ResponseField>

<ResponseField name="market_cap" type="float">
  Market capitalization value parsed from Bloomberg data (may include K/M/B/T suffixes).
</ResponseField>

<ResponseField name="shares_outstanding" type="float">
  Total shares outstanding parsed from Bloomberg data.
</ResponseField>

<ResponseField name="founded_year" type="int">
  Year the company was founded, extracted from Bloomberg data.
</ResponseField>

<ResponseField name="currency" type="string">
  Currency code for the company's fundamental data (e.g., "SGD", "HKD").
</ResponseField>

<ResponseField name="fetch_ok" type="boolean">
  Indicates whether the Bloomberg API request was successful. Only records with `fetch_ok=true` are written to the database.
</ResponseField>

<ResponseField name="fetched_at_utc" type="string">
  ISO timestamp when data was fetched (added by write_table).
</ResponseField>

### Data Source

Fetched using `fetch_bloomberg_equities()` from `BloombergSPCompanyFetcher`.

### Data Processing

1. **Deduplication**: Input companies are deduplicated by symbol (case-insensitive)
2. **MIC Codes**: 
   - SGX companies → `"XSES"` MIC code
   - HKEX companies → `"XHKG"` MIC code
3. **Ticker Generation**: Creates Bloomberg ticker format (symbol + ":" + suffix, e.g., "D05:SP", "0700:HK")
4. **Filtering**: Only records with `fetch_ok=true` are saved to the database

### Example Record

```json
{
  "symbol": "D05",
  "name": "DBS Group Holdings Ltd",
  "mic": "XSES",
  "bloomberg_ticker": "D05:SP",
  "source_url": "https://www.bloomberg.com/markets2/api/quote/EQT/D05%3ASP",
  "market_cap": 95234567890.0,
  "shares_outstanding": 2658000000.0,
  "founded_year": 1968,
  "currency": "SGD",
  "fetch_ok": true,
  "fetched_at_utc": "2026-03-03T10:30:00.123456+00:00"
}
```

### Usage

```python
from load_exchange_metadata_mysql import (
    fetch_sgx_companies,
    fetch_hkex_companies,
    fetch_bloomberg_equities,
    DEFAULT_TABLES
)

# Fetch source data
sgx_rows = fetch_sgx_companies(enrich=True)
hkex_rows = fetch_hkex_companies(enrich=True)

# Fetch Bloomberg data
sgx_bloomberg, hk_bloomberg, all_bloomberg, debug = fetch_bloomberg_equities(
    sgx_companies=sgx_rows,
    hkex_companies=hkex_rows,
    max_workers=1,
    delay_per_request=0.5,
    request_timeout=20
)

# Filter to successful fetches only
successful_records = [r for r in all_bloomberg if r.get("fetch_ok")]

print(f"Table: {DEFAULT_TABLES['bloomberg_equities']}")
print(f"Total fetched: {len(all_bloomberg)}")
print(f"Successful: {len(successful_records)}")
print(f"Sample: {successful_records[0] if successful_records else 'No records'}")
```

## Object Serialization

The `write_table()` function automatically serializes complex data types:

```python
def serialize_objects(df: pd.DataFrame) -> pd.DataFrame:
    for col in df.columns:
        if df[col].dtype == "object":
            df[col] = df[col].map(
                lambda v: json.dumps(v, ensure_ascii=False)
                if isinstance(v, (dict, list))
                else v
            )
    return df
```

### Behavior

- **dict** and **list** objects are serialized to JSON strings using `json.dumps()`
- Uses `ensure_ascii=False` to preserve Unicode characters
- Other object types are left unchanged
- Applied automatically before writing to the database

### Example

```python
# Before serialization
record = {
    "symbol": "0700",
    "executives": [{"name": "Ma Huateng", "title": "Chairman"}],
    "metadata": {"founded": 1998}
}

# After serialization (in database)
# executives: '[{"name": "Ma Huateng", "title": "Chairman"}]'
# metadata: '{"founded": 1998}'
```

## Database Configuration

The default database connection uses MySQL with the PyMySQL driver:

```python
DEFAULT_DB_URL = (
    "mysql+pymysql://user:password@host:port/database?charset=utf8mb4"
)
```

### Connection String Format

```
mysql+pymysql://[user]:[password]@[host]:[port]/[database]?charset=utf8mb4
```

### Character Set

The `charset=utf8mb4` parameter ensures proper handling of:
- International characters
- Emojis
- Extended Unicode characters

### Creating Engine

```python
from sqlalchemy import create_engine

db_url = "mysql+pymysql://user:pass@localhost:3306/MarketDataHub?charset=utf8mb4"
engine = create_engine(db_url)
```

## Write Modes

The `if_exists` parameter in `write_table()` controls table behavior:

### Replace Mode

```python
write_table(rows, "companies_xses", engine, if_exists="replace", fetched_at_utc=timestamp)
```

- Drops the existing table
- Creates a new table with fresh schema
- Inserts all rows
- **Use case**: Full data refresh

### Append Mode

```python
write_table(rows, "companies_xses", engine, if_exists="append", fetched_at_utc=timestamp)
```

- Keeps existing table and data
- Inserts new rows at the end
- **Use case**: Incremental updates or historical tracking
- **Note**: May create duplicates if the same data is appended multiple times
