## Overview

The SGX data source fetches comprehensive company information from the Singapore Exchange official APIs, including financial metrics, sector classifications, and trading status.

<Info>
  **Data Source**: Official SGX API at `api.sgx.com`
  
  **Coverage**: ~600 active companies on SGX mainboard and Catalist
</Info>

## API Endpoints

The SGX fetcher uses two primary endpoints:

### Stock Screener API

Provides financial metrics and basic company information:

```python
SCREENER_URL = "https://api.sgx.com/stockscreener/v2.0/all"
```

**Parameters fetched:**
- Market capitalization, P/E ratio, price-to-book ratio
- Dividend yield, sales TTM, net profit margin
- Return on equity, debt-to-equity ratio
- Price changes (4-week, 13-week, 26-week, 52-week)
- Basic sector classification

### Securities API

Provides trading status and suspension information:

```python
SECURITIES_URL = "https://api.sgx.com/securities/v1.1"
```

**Used for:**
- Identifying suspended securities (status = "SUSP")
- Validating active trading status

### Snapshot Reports API

Provides detailed sector and industry classifications:

```python
SNAPSHOT_URL = "https://api.sgx.com/snapshotreports/v2.0/countryCode/SGP/stockCode/{symbol}"
```

## Filtering Logic

The SGX fetcher applies automatic filtering:

```python
# From sgx_company_list.py:289-314
companies = fetcher.get_company_list(include_null_companies=False)

initial_count = len(companies)

# Filter out suspended companies
active_companies = [c for c in companies if not c["is_suspended"]]
suspended_count = initial_count - len(active_companies)

print(f"Initial count: {initial_count}")
print(f"Removed (suspended): {suspended_count}")
print(f"Final count: {len(active_companies)}")
```

<Note>
  By default, `include_null_companies=False` excludes companies with null exchange or company name.
</Note>

## Enrichment Process

The enrichment process fetches detailed sector data for each company:

```python
# From sgx_company_list.py:153-213
def enrich_with_sector_details(
    self,
    companies: list[dict],
    max_workers: int = 5,
    delay_per_request: float = 0.05,
    progress_callback=None
) -> list[dict]:
    """
    Enrich company list with detailed sector/industry info from snapshot reports.
    
    Args:
        companies: List of company dictionaries to enrich
        max_workers: Number of parallel workers
        delay_per_request: Delay between requests to avoid rate limiting
        progress_callback: Optional callback function for progress updates
    
    Returns:
        Enriched list of companies
    """
    from concurrent.futures import ThreadPoolExecutor, as_completed
    
    total = len(companies)
    completed = 0
    lock = __import__('threading').Lock()
    
    def fetch_and_enrich(company: dict) -> dict:
        nonlocal completed
        
        # Add small delay to avoid rate limiting
        time.sleep(delay_per_request)
        
        symbol = company["symbol"]
        details = self.fetch_snapshot_report(symbol)
        
        if details:
            company.update({
                "industry": details.get("sectorName"),
                "sector": details.get("industryName"),
            })
        
        with lock:
            completed += 1
            if progress_callback:
                progress_callback(completed, total)
        
        return company
    
    # Use ThreadPoolExecutor with rate limiting
    with ThreadPoolExecutor(max_workers=max_workers) as executor:
        futures = {executor.submit(fetch_and_enrich, c): c for c in companies}
        enriched = []
        for future in as_completed(futures):
            enriched.append(future.result())
    
    # Sort back by symbol
    enriched.sort(key=lambda x: x["symbol"])
    
    return enriched
```

<Warning>
  Enrichment makes additional API calls for each company. For 600 companies with 50ms delay, expect ~30 seconds processing time.
</Warning>

## Rate Limiting

Recommended rate limiting configuration:

```python
# Conservative (recommended for production)
fetcher.enrich_with_sector_details(
    companies,
    max_workers=5,
    delay_per_request=0.05  # 50ms between requests
)

# Aggressive (use with caution)
fetcher.enrich_with_sector_details(
    companies,
    max_workers=10,
    delay_per_request=0.01  # 10ms between requests
)
```

<Warning>
  Aggressive rate limiting may trigger API throttling. Monitor for HTTP 429 errors.
</Warning>

```python
# From sgx_company_list.py:38-59
def fetch_stock_screener_data(self) -> Optional[dict]:
    """
    Fetch stock screener data from SGX API.
    
    Returns:
        Dictionary containing the stock screener response
    """
    params = {
        "params": "exchange,exchangeCountryCode,companyName,..."
    }
    
    try:
        response = self.session.get(self.SCREENER_URL, params=params, timeout=30)
        response.raise_for_status()
        return response.json()
    
    except requests.RequestException as e:
        print(f"Error fetching screener data: {e}")
        return None
    except json.JSONDecodeError as e:
        print(f"Error parsing response: {e}")
        return None
```
